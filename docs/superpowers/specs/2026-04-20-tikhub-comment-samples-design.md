# TikHub 评论样本注入设计

## 目标
在 `link` 模式下，通过 TikHub API 拉取视频评论样本，并在用户开启开关时，将这些原始评论样本注入提示词，让模型模仿评论区的语气、句式和节奏。

## 范围
- 仅影响 `link` 模式
- 仅在页面开关开启时拉取并注入评论样本
- `upload` 模式保持不变
- 失败时自动降级，不阻断生成

## 交互
在页面的 `link` 输入区增加一个开关：
- 注入评论样本
  - 开：解析链接后额外拉取评论样本，并在生成时注入 prompt
  - 关：只使用当前已有的标题/基础上下文

## 数据流
1. 用户输入抖音链接
2. 后端解析链接时拿到视频标题等信息
3. 当开关开启且模式为 `link` 时，后端再从 TikHub 拉取评论样本
4. 对样本做轻量清洗：
   - 去空白
   - 去掉空行
   - 截断过长项
   - 过滤明显重复/噪音
5. 将标题 + 评论样本拼入 `Context & Input`
6. 模型根据这些样本风格生成评论

## Prompt 注入原则
- 注入“原始评论样本”，而不是摘要
- 提示词明确说明：这是样本，不要照抄，只学习语气、句式和长度分布
- 样本数量保持少量，避免把 prompt 过度污染

## 回退策略
- TikHub 无样本 / 请求失败：不报错，直接退化为仅注入标题或基础上下文
- 样本为空：不影响生成
- 页面开关关闭：不请求评论样本
