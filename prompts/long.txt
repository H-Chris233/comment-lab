# Task

# Core Constraints
1. 类型混排（同长度下随机交叉，禁止单一化）：
   • 细节共鸣型：带场景细节，如"这个细节一出来，氛围感直接拉满了"
   • 疑问追问型：自然追问，如"后面那个衔接是怎么做的？好丝滑"
   • 情绪共鸣型：情绪+场景，如"看到这里突然想起我上次也有同款感受"
   • 轻微吐槽型：中立反馈，如"整体挺顺，但结尾稍微有点仓促"
   • 路人闲聊型：适度发散，如"主包这段状态好自然，越看越舒服"（占比≈10%）
2. 表达多样性：句式/语气/观点必须打乱，禁止连续同质化或排比式输出。
3. 网感与符号：可少量穿插"真实住了""狠狠懂了"[笑哭]"…"等，位置随机，总量≤15%。
4. 允许少量反问，句末可用？或！
5. emoji：仅可少量加入以下 emoji：😎🌹😡👋😅😂😲👍😣🤣🥀。其中约 40% 的 emoji 放在句首；约 10% 的评论可连续放 3 个 emoji（如"😂😂😂"），其余位置随机，总量≤5%，不要连续堆砌。
6. 尽量避免"被种草了"、"博主"这类营销化或指代模糊表达；可改为"有点心动了""想试试"等更自然说法。
7. 禁止出现营销化/跟风化表达、测评腔/实测腔/结果总结腔、广告对比腔，以及讲拍摄手法、镜头语言、运镜、剪辑、构图、机位、转场、帧率、开头片段等内容的评价；也不要出现“允许泛化”“前面更好看”“笑死我了”“有点上头啊”“刷到就看完”“怎么拍的？”这类句式。
8. 输出格式：每条评论独占一行，纯文本，无编号/无标题/无引导语/无解释。
